# 젬베 리듬 게임: 난이도별 리듬 선택 메뉴 개발 바이브코딩 프롬프트

이 문서는 기존 **젬베 리듬 게임 프로토타입**에 “다양한 젬베 리듬을 난이도별로 선택하는 메뉴”를 추가하기 위한 개발 프롬프트입니다. Cursor, Claude Code, ChatGPT Codex, Replit Agent, Bolt, Lovable 같은 코딩 에이전트에 그대로 붙여 넣어 사용할 수 있도록 작성했습니다.

---

## 0. 개발 목표 요약

기존 시작 화면은 “게임 시작 / 연습 모드 / 설정 / 조작법” 중심으로 구성되어 있다. 이제 “게임 시작”을 누르면 바로 게임이 시작되는 구조가 아니라, 사용자가 **입문 / 초급 / 중급 / 고급** 난이도별로 다양한 젬베 리듬을 고르고, 미리듣기와 연습 모드를 거친 뒤 플레이할 수 있게 만든다.

핵심 추가 기능은 다음과 같다.

1. **리듬 선택 화면 또는 모달** 추가
2. **난이도별 리듬 카드** 추가
3. 각 리듬의 **BPM, 길이, 노트 수, 난이도, 추천 대상, 최고 기록** 표시
4. 리듬 카드 선택 시 **상세 정보 패널** 표시
5. **미리듣기** 기능 추가
6. 선택한 리듬으로 **게임 시작**
7. 선택한 리듬으로 **연습 모드 시작**
8. 결과창에 **선택 리듬명, 난이도, 최고 기록, 다음 추천 리듬** 표시
9. localStorage로 **최근 선택 리듬, 리듬별 최고 점수, 최고 콤보, 최고 정확도, 최고 등급** 저장
10. 기존 Slap/Bass/Tone, D/Space/K, 모바일 터치, AudioContext, 점수/콤보, 결과창, 카운트다운, 파티클 기능을 절대 깨지 않기

## 1. 코딩 에이전트에 붙여 넣을 메인 프롬프트

```text
너는 GPT-5.5급 기획력, UX 설계력, HTML5 Canvas 게임 개발력, Web Audio API 최적화 능력을 갖춘 시니어 웹 리듬게임 개발자다.

지금부터 기존 “젬베 리듬” HTML5 Canvas 리듬게임 프로토타입에 “난이도별 리듬 선택 메뉴”를 추가해라.

기존 게임의 핵심 구조는 유지해야 한다.
- 제목: 젬베 리듬
- 입력: Slap = D, Bass = Space, Tone = K
- 모바일 입력: 화면 왼쪽/가운데/오른쪽 터치
- Canvas 기반 노트 낙하
- Web Audio API / AudioContext 기반 저지연 사운드
- Score / Combo / Accuracy / Result
- 3초 카운트다운
- 노트 히트 파티클
- Slap/Bass/Tone 레인 겹침 방지
- 노트 간격 넉넉하게 유지
- 이미지와 사운드 에셋 fallback 유지

이번 작업의 목표는 “게임 시작” 버튼을 누르면 바로 플레이가 시작되는 단순 구조에서 벗어나, 사용자가 여러 젬베 리듬을 난이도별로 선택하고 플레이할 수 있게 만드는 것이다.

질문하지 말고, 아래 요구사항을 모두 반영해서 가장 완성도 높은 방식으로 구현해라.
```

## 2. 새 사용자 흐름 요구사항

```text
사용자 흐름을 다음처럼 바꿔라.

1. STATE_READY
   - 현재 시작 화면을 유지한다.
   - 큰 “게임 시작” 버튼의 동작을 바꾼다.
   - 기존: 게임 시작 → countdown → playing
   - 변경: 게임 시작 → STATE_RHYTHM_SELECT

2. STATE_RHYTHM_SELECT
   - 새 리듬 선택 화면 또는 전체 화면 모달을 띄운다.
   - 제목: “리듬 선택”
   - 서브카피: “난이도에 맞는 젬베 리듬을 골라보세요”
   - 상단에 난이도 탭: 전체 / 입문 / 초급 / 중급 / 고급
   - 중앙에 리듬 카드 목록
   - 하단 또는 우측에 선택한 리듬 상세 패널
   - 버튼: “미리듣기”, “연습하기”, “이 리듬으로 시작”
   - 뒤로가기 버튼: 시작 화면으로 돌아가기

3. 리듬 카드 선택
   - 선택된 카드가 강조된다.
   - 상세 패널에 해당 리듬의 설명, BPM, 길이, 노트 수, 난이도, 팁, 최고 기록을 표시한다.
   - 작은 3레인 패턴 미리보기 그래픽을 그린다.

4. “미리듣기” 클릭
   - AudioContext를 resume/init 한다.
   - 선택한 리듬의 앞 1~2마디를 Slap/Bass/Tone 사운드로 재생한다.
   - 미리듣기 중에는 카드 또는 미니 패턴에서 현재 박자를 하이라이트한다.
   - 다시 누르면 미리듣기를 중지한다.

5. “연습하기” 클릭
   - 선택한 리듬으로 연습 모드를 시작한다.
   - 연습 모드는 노트를 더 천천히 보여주고 판정창을 넓힌다.
   - 결과창은 나오되, 기록 저장은 “연습 기록”으로 따로 저장하거나 최고 기록에는 반영하지 않는다.

6. “이 리듬으로 시작” 클릭
   - 선택한 리듬을 currentChartId로 설정한다.
   - 3초 카운트다운 후 STATE_PLAYING으로 진입한다.

7. STATE_RESULT
   - 결과창에 선택한 리듬명과 난이도를 표시한다.
   - 리듬별 최고 점수/최고 콤보/최고 정확도/최고 등급을 갱신한다.
   - 다음 추천 리듬을 표시한다.
   - 버튼: “같은 리듬 다시 하기”, “다른 리듬 선택”, “연습 모드”, “처음으로”
```

## 3. 새 게임 상태 정의

기존 상태 정의에 아래 상태를 추가해라.

```js
const GAME_STATES = {
  BOOT: "boot",
  READY: "ready",
  RHYTHM_SELECT: "rhythmSelect",
  COUNTDOWN: "countdown",
  PLAYING: "playing",
  PAUSED: "paused",
  PRACTICE: "practice",
  RESULT: "result",
  SETTINGS: "settings",
  HELP: "help"
};
```

상태 전환은 반드시 한 곳에서 관리해라.

```js
class GameStateManager {
  changeState(nextState, payload = {})
  getState()
  is(stateName)
}
```

핵심 전환:

```text
BOOT → READY
READY → RHYTHM_SELECT
RHYTHM_SELECT → COUNTDOWN
RHYTHM_SELECT → PRACTICE
COUNTDOWN → PLAYING
PLAYING → RESULT
PRACTICE → RESULT 또는 RHYTHM_SELECT
RESULT → COUNTDOWN
RESULT → RHYTHM_SELECT
RESULT → READY
```

패턴 표기 규칙:

```text
"bass" 또는 "B" = Bass 노트
"tone" 또는 "T" = Tone 노트
"slap" 또는 "S" = Slap 노트
"-" 또는 null = 쉼표
```

patternUnit 변환:

```text
quarter: 한 칸 = 1 beat
 eighth: 한 칸 = 0.5 beat
sixteenth: 한 칸 = 0.25 beat
```

초보자용 게임이므로 sixteenth를 쓰더라도 실제 노트 간격 검증에서 너무 가까우면 자동으로 쉬운 간격으로 벌려야 한다.

## 8. 차트 생성 로직

기존 하드코딩 차트만 쓰지 말고, 리듬 데이터의 bars를 note 배열로 변환하라.

```js
function normalizeLaneToken(token) {
  if (!token || token === "-") return null;
  const t = String(token).toLowerCase();
  if (t === "b" || t === "bass") return "bass";
  if (t === "t" || t === "tone") return "tone";
  if (t === "s" || t === "slap") return "slap";
  return null;
}

function getBeatStep(patternUnit) {
  if (patternUnit === "quarter") return 1;
  if (patternUnit === "eighth") return 0.5;
  if (patternUnit === "sixteenth") return 0.25;
  return 0.5;
}

function beatToMs(beat, bpm) {
  return beat * (60000 / bpm);
}

function generateNotesFromRhythm(rhythm) {
  const notes = [];
  const stepBeat = getBeatStep(rhythm.patternUnit);
  let noteIndex = 0;
  let beatCursor = 4; // 시작 전 4박 여유. 카운트다운 후 첫 노트가 너무 빨리 나오지 않게 한다.

  for (let r = 0; r < rhythm.repeat; r++) {
    for (const bar of rhythm.bars) {
      for (let i = 0; i < bar.length; i++) {
        const lane = normalizeLaneToken(bar[i]);
        if (lane) {
          notes.push({
            id: `${rhythm.id}_${noteIndex++}`,
            lane,
            beat: beatCursor + i * stepBeat,
            hitTimeMs: beatToMs(beatCursor + i * stepBeat, rhythm.bpm),
            status: "pending"
          });
        }
      }
      beatCursor += bar.length * stepBeat;
    }
  }

  return notes;
}
```

## 9. 노트 겹침 방지 검증

리듬 선택 기능이 추가되어도 기존 문제였던 “노트 겹침”이 절대 다시 생기면 안 된다.

```js
function validateAndFixChartSpacing(chart, rhythm) {
  const minGlobalGapMs = rhythm.minGlobalGapMs || DIFFICULTY_CONFIG[rhythm.difficulty].minGlobalGapMs;
  const minSameLaneGapMs = rhythm.minSameLaneGapMs || DIFFICULTY_CONFIG[rhythm.difficulty].minSameLaneGapMs;

  chart.notes.sort((a, b) => a.hitTimeMs - b.hitTimeMs);

  let lastAnyTime = -Infinity;
  const lastLaneTime = {
    slap: -Infinity,
    bass: -Infinity,
    tone: -Infinity
  };

  for (const note of chart.notes) {
    const gapAny = note.hitTimeMs - lastAnyTime;
    const gapLane = note.hitTimeMs - lastLaneTime[note.lane];

    let requiredShift = 0;
    if (gapAny < minGlobalGapMs) {
      requiredShift = Math.max(requiredShift, minGlobalGapMs - gapAny);
    }
    if (gapLane < minSameLaneGapMs) {
      requiredShift = Math.max(requiredShift, minSameLaneGapMs - gapLane);
    }

    if (requiredShift > 0) {
      note.hitTimeMs += requiredShift;
      note.autoShifted = true;
    }

    lastAnyTime = note.hitTimeMs;
    lastLaneTime[note.lane] = note.hitTimeMs;
  }

  chart.durationMs = Math.max(
    rhythm.durationMs || 0,
    chart.notes.length ? chart.notes[chart.notes.length - 1].hitTimeMs + 3000 : 30000
  );

  return chart;
}
```

```js
function validateNoSameTimeNotes(chart) {
  const seen = new Map();
  for (const note of chart.notes) {
    const key = Math.round(note.hitTimeMs / 10) * 10;
    if (seen.has(key)) {
      note.hitTimeMs += 180;
      note.autoShifted = true;
    }
    seen.set(key, true);
  }
}
```

## 14. 미니 패턴 프리뷰 Canvas

선택한 리듬의 앞 1~2마디를 작은 Canvas에 그려라.

요구사항:
- 세 줄: Slap, Bass, Tone
- 왼쪽에 라벨 표시
- 노트는 lane별 작은 원으로 표시
- Bass는 큰 원, Tone은 중간 원, Slap은 별 또는 작은 번쩍임 아이콘처럼 표시해도 좋다.
- 현재 미리듣기 위치는 세로 라인으로 표시
- Reduced Motion이 켜져 있으면 커서 애니메이션만 단순하게 처리

의사 코드:

```js
function drawPatternPreview(canvas, rhythm, previewProgress = 0) {
  const ctx = canvas.getContext("2d");
  const w = canvas.width;
  const h = canvas.height;
  ctx.clearRect(0, 0, w, h);

  const rows = {
    slap: h * 0.25,
    bass: h * 0.52,
    tone: h * 0.79
  };

  // 배경 라인
  Object.entries(rows).forEach(([lane, y]) => {
    ctx.globalAlpha = 0.5;
    ctx.beginPath();
    ctx.moveTo(54, y);
    ctx.lineTo(w - 12, y);
    ctx.stroke();
    ctx.globalAlpha = 1;
    ctx.fillText(lane.toUpperCase(), 8, y + 4);
  });

  const notes = generatePreviewNotes(rhythm, rhythm.previewBars || 2);
  const maxBeat = getPreviewMaxBeat(rhythm, rhythm.previewBars || 2);

  for (const note of notes) {
    const x = 58 + (note.beat / maxBeat) * (w - 76);
    const y = rows[note.lane];
    drawMiniNote(ctx, x, y, note.lane);
  }

  // 커서
  const cursorX = 58 + previewProgress * (w - 76);
  ctx.beginPath();
  ctx.moveTo(cursorX, 10);
  ctx.lineTo(cursorX, h - 10);
  ctx.stroke();
}
```

## 15. 미리듣기 기능

AudioEngine을 활용해서 리듬의 앞 1~2마디를 실제 사운드로 들려줘라.

```js
class RhythmPreviewPlayer {
  constructor(audioEngine, rhythmLibrary, ui) {
    this.audioEngine = audioEngine;
    this.rhythmLibrary = rhythmLibrary;
    this.ui = ui;
    this.timerIds = [];
    this.playing = false;
    this.previewStartMs = 0;
    this.previewDurationMs = 0;
  }

  async playPreview(rhythmId) {
    this.stopPreview();
    await this.audioEngine.ensureStarted();

    const rhythm = this.rhythmLibrary.getRhythmById(rhythmId);
    const notes = generatePreviewNotes(rhythm, rhythm.previewBars || 2);
    const previewDurationMs = getPreviewDurationMs(rhythm, rhythm.previewBars || 2);

    this.playing = true;
    this.previewStartMs = performance.now();
    this.previewDurationMs = previewDurationMs;

    for (const note of notes) {
      const delay = note.hitTimeMs;
      const timerId = setTimeout(() => {
        if (!this.playing) return;
        this.audioEngine.play(note.lane, { gain: 0.8 });
        this.ui.flashPreviewNote(note);
      }, delay);
      this.timerIds.push(timerId);
    }

    const endTimer = setTimeout(() => this.stopPreview(), previewDurationMs + 200);
    this.timerIds.push(endTimer);

    this.animatePreviewCursor();
  }

  animatePreviewCursor() {
    if (!this.playing) return;
    const elapsed = performance.now() - this.previewStartMs;
    const progress = Math.min(1, elapsed / this.previewDurationMs);
    this.ui.updatePreviewCursor(progress);
    if (progress < 1) requestAnimationFrame(() => this.animatePreviewCursor());
  }

  stopPreview() {
    this.timerIds.forEach(id => clearTimeout(id));
    this.timerIds = [];
    this.playing = false;
    this.ui?.updatePreviewCursor?.(0);
  }
}
```

주의:
- 게임이 시작되면 preview는 반드시 중지한다.
- 설정창이나 뒤로가기 시 preview를 중지한다.
- 미리듣기 중 다른 리듬을 선택하면 이전 preview를 중지한다.

```js
function safeLocalStorageGet(key, fallback) {
  try {
    const raw = localStorage.getItem(key);
    return raw ? JSON.parse(raw) : fallback;
  } catch (error) {
    console.warn("localStorage get failed:", key, error);
    return fallback;

}
```

```text
- preview 중복 재생을 방지해라.
```

### 22.5 5단계: 연습하기 모드 연결

```text
선택한 리듬으로 연습 모드를 시작하는 기능을 추가해줘.

이번 단계에서 할 일:
1. “연습하기” 버튼 구현
2. startSelectedRhythm("practice") 구현
3. practice mode에서는 approachTimeMs를 1.18배로 늘림
4. judgement window를 1.25배로 넓힘
5. 필요하면 BPM을 0.88배로 낮춤
6. 결과창에는 “연습 모드” 배지 표시
7. practice 결과는 최고 기록에는 반영하지 않음
8. 결과창에서 “연습 모드 다시 하기”와 “일반 모드 도전” 버튼 제공

중요:
- 자유 연습 모드가 이미 있다면, 이 기능은 “선택 리듬 연습”으로 구분해라.
```

## 23. 코딩 에이전트가 반드시 지켜야 할 강제 규칙

```text
강제 규칙:

1. 기존 Slap/Bass/Tone 입력 구조를 절대 깨지 마라.
2. D = Slap, Space = Bass, K = Tone 매핑을 유지해라.
3. 모바일 왼쪽/가운데/오른쪽 터치 입력을 유지해라.
4. 리듬 선택 메뉴 추가 후에도 게임 시작 전 3초 카운트다운을 유지해라.
5. 모든 리듬 차트는 validateAndFixChartSpacing()을 통과해야 한다.
6. 노트가 같은 시간에 겹쳐 떨어지면 안 된다.
7. 같은 레인 연타는 각 난이도의 minSameLaneGapMs보다 가까우면 안 된다.
8. 서로 다른 레인도 minGlobalGapMs보다 가까우면 자동으로 벌려라.
9. “미리듣기” 중 실제 게임을 시작하면 preview를 반드시 중지해라.
10. 연습 모드 결과는 최고 기록에 저장하지 마라.
11. localStorage 오류가 나도 게임이 죽지 않아야 한다.
12. AudioContext는 사용자 클릭/터치 이후에만 resume/init 해라.
13. 이미지나 사운드 에셋 누락 시 fallback을 유지해라.
14. 메뉴 UI는 모바일에서 잘려서는 안 된다.
15. Space 키로 페이지가 스크롤되면 안 된다.
16. 모든 핵심 수치는 CONFIG 또는 DIFFICULTY_CONFIG에서 조절 가능해야 한다.
17. 기존 코드 전체를 불필요하게 갈아엎지 말고, 확장 방식으로 구현해라.
18. 최종 코드는 실행 가능한 상태로 출력해라.
```
